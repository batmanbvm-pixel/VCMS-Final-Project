## 🎓 VIVA Examination Tips

### Before Viva
1. **Get good sleep** - Don't cram the night before
2. **Review VIVA_QUICK_REFERENCE.md** - Quick 30-min refresh
3. **Have code ready** - Know where each file is located
4. **Have DATABASE.md open** - Reference schema during viva
5. **Have FLOW.md open** - Reference architecture diagrams

### During Viva
1. **Start with confidence** - Explain project overview clearly
2. **Use diagrams** - Reference FLOW.md architecture diagrams
3. **Show code** - Be ready to show specific implementations
4. **Explain flow** - Trace request from frontend to backend to database
5. **Admit limitations** - It's okay to say "I don't know, let me check"
6. **Ask for clarification** - If question is unclear, ask for clarification

### Common Viva Questions (All in VIVA_GUIDE.md)
- "What is MERN?"
- "Explain your project architecture"
- "How does authentication work?"
- "What is this API endpoint doing?"
- "How does this validation work?"
- "Why did you use this technology?"
